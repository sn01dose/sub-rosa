# SUB ROSA Character System v2

16タイプを「同じ絵師・同じシリーズの別人」として描き分けるための制作ガイドです。確定済み `FGPD` の頭身・線・水彩・金枠だけをシリーズ共通要素とし、人物の髪、衣装、ポーズ、表情、背景小景、紋章はタイプ別に変えます。生成時の機械可読な正本は `tools/character-specs.json` です。

## 1. 固定するもの

- 確定FGPDに合わせた、成人らしい2.2〜2.7頭身のちびシルエット。
- 繊細な暗色線画、にじみのある水彩、少量の紙目、くすみ色。
- 細いアンティークゴールドのアールヌーヴォーアーチ。
- アーチ内の薄いセピア小景。人物・枠・小景の外側は透過。
- 単独の全身人物、縦長構図、文字・署名・タイトルなし。

共通ブロックには服装、髪色、ポーズ、月、猫、書斎などを入れません。特に `dark attire`、黒ボブ、片目隠し、片足クロス、丸い黒コートはFGPD固有であり、シリーズ共通ではありません。

## 2. タイプごとに変えるもの

各タイプは次の7スロットを必ず持ちます。

1. `hair`: 色、長さ、結び方、外形。
2. `attire`: 服の種類と大きな面の形。
3. `gestureExpression`: 所作、視線、表情、手の役割。
4. `emblem`: アーチ上部に置く唯一の紋章。
5. `vignette`: アーチ内の薄い背景小景。
6. `silhouette`: サムネイルでも読める輪郭と重心。
7. `motif.color`: 衣装の25〜30%以上を占める主色。

小物だけを差し替えた同一人物を作らないため、髪の外形、衣装の最大面、手の高さ、重心のうち最低3項目を他タイプと変えます。

## 3. 4軸から見た目への文法

| 位置 | 軸 | 見た目への変換 |
|---|---|---|
| 1 | L | 背筋を立て、視線はまっすぐ。招く、示す、提示する手を輪郭に出す。 |
| 1 | F | 小首をかしげ、上目または伏し目。手は胸元、受け取る位置、後ろ手へ寄せる。 |
| 2 | G | 小物を差し出す、または道具を扱う手を主役にする。 |
| 2 | T | 受け取る、胸へ抱き寄せる動作を主役にする。 |
| 3 | R | 暖色アクセント、控えめな頬の赤み、花またはハートの微モチーフ。 |
| 3 | P | 寒色寄り、澄ました／不敵な表情、夜の空気または四光星の微モチーフ。 |
| 4 | C | 左右対称に近い輪郭、きっちり留めた服、静止した重心。 |
| 4 | D | 非対称、髪やコートの翻り、外衣の抜け、動きの途中の重心。 |

軸文法はタイプ別スペックを置き換えず、スペックに追記します。例としてFTRDは「Fの上目＋Tの受け取り＋Rの薔薇と頬染め＋Dの巻き髪と裾の流れ」を一枚に統合します。

## 5. 色と紋章

- `data/types.js` の `motif.color` を衣装の25〜30%以上へ使う。裏地、ベスト、帯、マント、コート胴など、遠目でも見える大面積に置く。
- 金は枠、留め具、紋章の細線に限定する。全員を黒いロングコートへ戻さない。
- アーチ上部はスペック指定の紋章だけにする。別タイプの月、猫、星を残さない。
- 五芒星、オカルト記号、錬金術記号、星座記号、ルーン、文字に見える飾りは禁止。
- FGPCの時計盤は数字も目盛りもない無地。LTPDの紙は音符・譜線・文字のない白紙。

## 6. 生成パイプライン

1. `data/types.js` から `name / catch / personality / motif.color` を読む。
2. `tools/character-specs.json` からタイプ固有スロットと4軸文法を読む。
3. 共通描画ブロック → タイプスロット → 4軸文法 → 色面積 → 禁止事項の順に合成する。
4. まず参照画像なしのImages Generationでパイロットを作る。
5. 頭身・線・水彩・金枠が確定FGPDから割れた場合のみ、確定FGPDを参照するImages Editへ切り替える。参照時も髪・衣装・ポーズ・紋章・小景はコピーしない。

## 7. 検品

`tools/out/contact.html` で確定FGPDと新規候補を並べ、通常サイズと約25%の遠目表示を切り替えます。

- 髪、衣装の外形、重心だけでタイプを識別できる。
- 4軸の姿勢・手・色・対称性が読める。
- `motif.color` が大きな衣装面へ出ている。
- 指、重複手足、文字、数字、音符、露出、幼い服装がない。
- 同じ画風だが別人に見える。

## 8. 書き出し

- 選定元: `tools/out/<output-set>/{TYPE_CODE}/candidate-{N}.png`。
- 主表示: `assets/characters/{TYPE_CODE}.webp`、2倍版: `{TYPE_CODE}@2x.webp`。
- 透明余白をトリムし、共通パディングで640×840 / 1280×1680へ正規化する。
- WebPが欠落した場合は同名SVG、さらに `assets/adult-silhouette.svg` へフォールバックする。
- 全身着衣、成人、中性的、非露骨。学校制服、下着、過度な露出、暴力、文字、透かしは禁止。
